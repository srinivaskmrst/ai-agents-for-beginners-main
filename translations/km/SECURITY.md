# សន្តិសុខ

Microsoft យកចិត្តទុកដាក់យ៉ាងខ្លាំងចំពោះសន្តិសុខនៃផលិតផល និងសេវាកម្មកម្មវិធីរបស់យើង ដែលរួមមានឃ្លាំងកូដដើមទាំងអស់ ដែលគ្រប់គ្រងតាមរយៈអង្គការរបស់យើងលើ GitHub ដែលរួមមាន [Microsoft](https://github.com/Microsoft), [Azure](https://github.com/Azure), [DotNet](https://github.com/dotnet), [AspNet](https://github.com/aspnet) និង [Xamarin](https://github.com/xamarin)។

បើអ្នកជឿថាអ្នកបានរកឃើញចំនុចខ្សោយសន្តិសុខនៅក្នុងឃ្លាំងទិន្ន័យដែលជាម្ចាស់ដោយ Microsoft ដែលត្រូវគ្នានឹង [និយមន័យសន្តិសុខរបស់ Microsoft](https://aka.ms/security.md/definition) សូមរាយការណ៍មកយើងដូចបានរៀបរាប់ខាងក្រោម។

## ការរាយការណ៏បញ្ហាសន្តិសុខ

**សូមកុំរាយការណ៍ចំណុចខ្សោយសន្តិសុខតាមរយៈបញ្ហាសាធារណៈនៅលើ GitHub។**

ផ្ទុយទៅវិញ សូមរាយការណ៍មកកាន់មជ្ឈមណ្ឌលតបទាញសន្តិសុខ Microsoft (MSRC) តាមរយៈ [https://msrc.microsoft.com/create-report](https://aka.ms/security.md/msrc/create-report)។

បើអ្នកចង់ដាក់ទំនាក់ទំនងដោយមិនចូលសូម ផ្ញើអ៊ីមែលទៅ [secure@microsoft.com](mailto:secure@microsoft.com)។ ប្រសិនបើអាចធ្វើបាន សូម mã hóa សាររបស់អ្នកជាមួយកូនសោ PGP របស់យើង; សូមទាញយកវាពីទំព័រ [Microsoft Security Response Center PGP Key](https://aka.ms/security.md/msrc/pgp)។

អ្នកគួរតែទទួលបានការឆ្លើយតបក្នុងរយៈពេល 24 ម៉ោង។ ប្រសិនបើដោយហេតុផលណាមួយ មិនទទួលបាន ការឆ្លើយតប សូមធ្វើការតាមដានតាមរយៈអ៊ីមែលដើម្បីធានាថាយើងទទួលបានសារដើមរបស់អ្នក។ ព័ត៌មានបន្ថែមអាចរកបាននៅ [microsoft.com/msrc](https://www.microsoft.com/msrc)។

សូមរួមបញ្ចូលព័ត៌មានដែលបានស្នើសុំខាងក្រោម (តាមដែលអាចផ្តល់បាន) ដើម្បីជួយយើងយល់ដឹងបានល្អជាងពីលក្ខណៈ និងវិសាលភាពនៃបញ្ហាដែលអាចកើតមាន:

* ប្រភេទបញ្ហា (ឧ. ការទាញយកអត្រាបញ្ចប់, ការចាក់បញ្ចូល SQL, ការចាក់ស្គ្រីបត់លើវេបសាយផ្សេងគ្នា, ល។)
* ផ្លូវពេញលេញនៃឯកសារមূলដើម្បីបង្ហាញបញ្ហា
* ទីតាំងកូដដើមដែលបានប៉ះពាល់ (tag/branch/commit ឬ URL ត្រង់)
* ការកំណត់ពិសេសណាមួយដែលចាំបាច់ដើម្បីធ្វើវិញបញ្ហា
* ជំហានជាក់លាក់ដើម្បីធ្វើវិញបញ្ហា
* កូដបង្ហាញការបង្កើតបញ្ហា ឬកូដប្រើប្រាស់ (ប្រសិនបើអាចបាន)
* ផលប៉ះពាល់នៃបញ្ហា រួមទាំងរបៀបដែលអ្នកចាប់កំហុសអាចប្រើប្រាស់បញ្ហាទាំងនេះ

ព័ត៌មាននេះនឹងជួយឲ្យយើងពិនិត្យរបាយការណ៍របស់អ្នកបានប្រសើរឡើង។

បើអ្នកកំពុងរាយការណ៍សម្រាប់រង្វាន់បន្ថែមលើកំហុស ការរាយការណ៍លម្អិតដំណើរៗនេះអាចជួយឲ្យទទួលបានរង្វាន់ខ្ពស់ជាង។ សូមចូលទៅកាន់ទំព័រ [Microsoft Bug Bounty Program](https://aka.ms/security.md/msrc/bounty) ដើម្បីពិនិត្យព័ត៌មានលម្អិតអំពីកម្មវិធីសកម្មរបស់យើង។

## ភាសាដែលចូលចិត្ត

យើងចូលចិត្តឲ្យការទំនាក់ទំនងទាំងអស់ធ្វើឡើងជាភាសាអង់គ្លេស។

## នយោបាយ

Microsoft អនុវត្តតាមគោលការណ៍ [Coordinated Vulnerability Disclosure](https://aka.ms/security.md/cvd)៕
