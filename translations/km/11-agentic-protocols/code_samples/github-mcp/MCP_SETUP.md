# មគ្គុទេសក៍រួមបញ្ចូល MCP Server

## តម្រូវការ
- Node.js ត្រូវបានដំឡើង (កំណែ 14 ឬខ្ពស់ជាងនេះ)
- npm (កម្មវិធីគ្រប់គ្រងកញ្ចប់)
- បរិយាកាស Python ដែលមានការពឹងផ្អែកដែលចាំបាច់

## ជំហានរៀបចំ

1. **តំឡើងកញ្ចប់ MCP Server**
   ```bash
   npm install -g @modelcontextprotocol/server-github
   ```

2. **ចាប់ផ្ដើម MCP Server**
   ```bash
   npx @modelcontextprotocol/server-github
   ```
   ម៉ាស៊ីនមេគួរតែចាប់ផ្ដើម ហើយបញ្ចាំង URL សម្រាប់ការតភ្ជាប់។

3. **ផ្ទៀងផ្ទាត់ការតភ្ជាប់**
   - សូមស្វែងរករូបសញ្ញា (🔌) នៅក្នុងអន្តរមុខ Chainlit របស់អ្នក
   - លេខ (1) គួរតែបង្ហាញនៅក្បែររូបសញ្ញាផ្លក់ ដើម្បីបង្ហាញពីការតភ្ជាប់ដោយជោគជ័យ
   - កុងសូលគួរតែបង្ហាញ៖ "ការតម្លើង plugin របស់ GitHub បានសម្រេចដោយជោគជ័យ" (រួមទាំងបន្ទាត់ស្ថានភាពបន្ថែម)

## ការដោះស្រាយបញ្ហា

### បញ្ហាធម្មតា

1. **ជម្លោះនៃ Port**
   ```bash
   Error: listen EADDRINUSE: address already in use
   ```
   ដំណោះស្រាយ៖ ប្ដូរច្រក (port) ដោយប្រើ៖
   ```bash
   npx @modelcontextprotocol/server-github --port 3001
   ```

2. **បញ្ហាការផ្ទៀងផ្ទាត់**
   - ប្រាកដថាព័ត៌មានអត្តសញ្ញាណ GitHub ត្រូវបានកំណត់ត្រឹមត្រូវ
   - ពិនិត្យឯកសារ .env ថាមាន tokens ត្រូវការ
   - ផ្ទៀងផ្ទាត់ការចូលប្រើ GitHub API

3. **ការតភ្ជាប់បរាជ័យ**
   - បញ្ជាក់ថាម៉ាស៊ីនមេកំពុងដំណើរការនៅលើច្រកដែលរំពឹង
   - ពិនិត្យការកំណត់ firewall
   - ផ្ទៀងផ្ទាត់ថាបរិយាកាស Python មានកញ្ចប់ដែលត្រូវការ

## ផ្ទៀងផ្ទាត់ការតភ្ជាប់

ម៉ាស៊ីនមេ MCP របស់អ្នកត្រូវបានតភ្ជាប់បានត្រឹមត្រូវពេល៖
1. កុងសូលបង្ហាញ៖ "ការតម្លើង plugin របស់ GitHub បានសម្រេចដោយជោគជ័យ"
2. កំណត់ហេតុការតភ្ជាប់បង្ហាញ៖ "✓ ស្ថានភាពតភ្ជាប់ MCP: សកម្ម"
3. ពាក្យបញ្ជា GitHub ធ្វើការបានក្នុងអន្តរមុខសន្ទនា

## អថេរបរិយាកាស

ត្រូវការនៅក្នុងឯកសារ .env របស់អ្នក:
```
GITHUB_TOKEN=your_github_token
MCP_SERVER_PORT=3000  # Optional, default is 3000
```

## សាកល្បងការតភ្ជាប់

ផ្ញើសារសាកល្បងនេះក្នុងសន្ទនា:
```
Show me the repositories for username: [GitHub Username]
```
ការឆ្លើយតបដោយជោគជ័យនឹងបង្ហាញព័ត៌មាននៃ repository.
