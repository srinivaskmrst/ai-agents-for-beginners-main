# 🔍 RAG សម្រាប់សហគ្រាស ជាមួយ Azure AI Foundry (.NET)

## 📋 គោលបំណងការរៀន

សរសេរណូតប៊ុកនេះបង្ហាញពីរបៀបសាងសង់ប្រព័ន្ធ Retrieval-Augmented Generation (RAG) វិសាយសម្រាប់សហគ្រាស ដោយប្រើ Microsoft Agent Framework នៅក្នុង .NET ជាមួយ Azure AI Foundry។ អ្នកនឹងរៀនសូត្របង្កើតភ្នាក់ងារឈឹងត្រៀមសម្រាប់ផលិតកម្ម ដែលអាចស្វែងរកឯកសារនានា និងផ្តល់ចម្លើយដែលត្រឹមត្រូវ មានបរិបទ និងមានសុវត្ថិភាពនិងអាចពង្រីកបានសម្រាប់សហគ្រាស។

**សមត្ថភាព RAG សម្រាប់សហគ្រាស ដែលអ្នកនឹងសាងសង់៖**
- 📚 **បញ្ញាឯកសារ**: ការកាត់ច្រោះឯកសារដែលទាន់សម័យជាមួយសេវាកម្ម Azure AI
- 🔍 **ការស្វែងរកដោយអត្ថន័យ**: ការស្វែងរកវ៉ិចទ័រក្នុងប្រសិទ្ធភាពខ្ពស់ ជាមួយលក្ខណៈសម្រាប់សហគ្រាស
- 🛡️ **ការរួមបញ្ចូលសុវត្ថិភាព**: ការចូលប្រើដោយផ្អែកលើតួនាទី និងលំនាំការពារទិន្នន័យ
- 🏢 **ស្ថាបត្យកម្មអាចពង្រីកបាន**: ប្រព័ន្ធ RAG រួចរាលសម្រាប់ផលិតកម្ម ជាមួយការត្រួតពិនិត្យ

## 🎯 ស្ថាបត្យកម្ម RAG សម្រាប់សហគ្រាស

### សមាសភាគសំខាន់សម្រាប់សហគ្រាស
- **Azure AI Foundry**: វេទិកា AI រដ្ឋបាលសម្រាប់សហគ្រាស អាចផ្តល់សុវត្ថិភាព និងការអនុវត្តតាមស្ដង់ដារ
- **ភ្នាក់ងារមានស្ថិតិភាព**: ភ្នាក់ងារស្ថិតិភាពជាមួយប្រវត្តិការពិភាក្សា និងការគ្រប់គ្រងបរិបទ
- **ការគ្រប់គ្រងហាងទិន្នន័យវ៉ិចទ័រ**: ការកំណត់ធាតុសម្រាប់ឯកសារនិងការយកវិញក្នុងកម្រិតសហគ្រាស
- **ការរួមបញ្ចូលអត្តសញ្ញាណ**: ការផ្ទៀងផ្ទាត់ Azure AD និងការគ្រប់គ្រងចូលប្រើដោយផ្អែកលើតួនាទី

### អត្ថប្រយោជន៍ .NET សម្រាប់សហគ្រាស
- **សុវត្ថិភាពប្រភេទ (Type Safety)**: ការផ្ទៀងផ្ទាត់នៅពេលកុំផៃលសម្រាប់ប្រតិបត្តិការនិងទិន្នន័យ RAG
- **ប្រសិទ្ធភាពអស៊ីនក្រូន**: ការរីករាលដាលដោយមិនរារាំងសម្រាប់ដំណើរការ និងការស្វែងរកឯកសារ
- **ការគ្រប់គ្រងអង្គចងចាំ**: ការប្រើប្រាស់ធនធានប្រកបដោយប្រសិទ្ធភាពសម្រាប់បណ្ណាល័យឯកសារធំៗ
- **លំនាំការរួមបញ្ចូល**: ការរួមបញ្ចូលសេវាកម្ម Azure ដោយដើមជាមួយ dependency injection

## 🏗️ ស្ថាបត្យកម្មបច្ចេកទេស

### លំហូរ RAG សម្រាប់សហគ្រាស
```
Document Upload → Security Validation → Vector Processing → Index Creation
                      ↓                    ↓                  ↓
User Query → Authentication → Semantic Search → Context Ranking → AI Response
```

### សមាសភាគសំខាន់ៗ ក្នុង .NET
- **Azure.AI.Agents.Persistent**: ការគ្រប់គ្រងភ្នាក់ងារ​សម្រាប់សហគ្រាស ជាមួយការរក្សាស្ថានភាព
- **Azure.Identity**: ការផ្គុំការផ្ទៀងផ្ទាត់រួមសម្រាប់ចូលប្រើសេវាកម្ម Azure មានសុវត្ថិភាព
- **Microsoft.Agents.AI.AzureAI**: ការអនុវត្តស៊ុមភ្នាក់ងារដែលឱ្យសមស្របសម្រាប់ Azure
- **System.Linq.Async**: ប្រតិបត្តិការលីញអស៊ីនក្រូនដែលមានប្រសិទ្ធភាពខ្ពស់

## 🔧 លក្ខណៈពិសេស & អត្ថប្រយោជន៍សម្រាប់សហគ្រាស

### សុវត្ថិភាព និងការអនុវត្តតាមនីតិវិធី
- **ការរួមបញ្ចូល Azure AD**: ការគ្រប់គ្រងអត្តសញ្ញាណសហគ្រាស និងការផ្ទៀងផ្ទាត់
- **ការចូលប្រើដោយផ្អែកលើតួនាទី**: អាជ្ញាបណ្ណលម្អិតសម្រាប់ការចូលប្រើឯកសារ និងប្រតិបត្តិការ
- **ការការពារទិន្នន័យ**: ការអ៊ិនគ្រីបនៅពេលឈប់ស្ងាត់ និងកំឡុងផ្ទេរក្នុងការពារឯកសារមានសេចក្ដីសំខាន់
- **កំណត់ហេតុត្រួតពិនិត្យ**: ការតាមដានសកម្មភាពយ៉ាងទូលំទូលាយសម្រាប់តម្រូវការអនុលោម

### ប្រសិទ្ធភាព និងការអាចពង្រីកបាន
- **ការបង្វិលតំណភ្ជាប់**: ការគ្រប់គ្រងតំណភ្ជាប់ទៅសេវាកម្ម Azure ដោយមានប្រសិទ្ធភាព
- **ដំណើរការអស៊ីនក្រូន**: ប្រតិបត្តិការ​មិនរារាំងសម្រាប់ស្ថានភាព throughput ខ្ពស់
- **យុទ្ធសាស្រ្ត caching**: ការកែច្នៃ caching យល់ដឹងសម្រាប់ឯកសារ​ដែលចូលដំណើរការតាមសារញឹកញាប់
- **ការផ្លាស់ប្តូរហារម៉ៅ**: ប្រតិបត្តិការចែកចាយសម្រាប់ដាក់បញ្ចូលនៅស្ដុកទំហំធំ

### ការគ្រប់គ្រង និងការត្រួតពិនិត្យ
- **ការត្រួតពិនិត្យសុខភាព**: ការត្រួតពិនិត្យដែលបានបង្កប់សម្រាប់សមាសភាគប្រព័ន្ធ RAG
- **មេត្រីកប្រសិទ្ធភាព**: វិភាគលម្អិតលើគុណភាពស្វែងរក និងពេលផ្លាស់បម្លាស់ចម្លើយ
- **ការដោះស្រាយកំហុស**: ការគ្រប់គ្រងករណីចាប់ពិសេសយ៉ាងទូលំទូលាយជាមួយយុទ្ធសាស្រ្តព្យាករ

## 📊 លំនាំ RAG សម្រាប់សហគ្រាស

### លំនាំគ្រប់គ្រងឯកសារ
- **ការដាក់ឡើងបន្ទុកច្រើន (Bulk Upload)**: ការបញ្ចូលដំណើរការដ៏មានប្រសិទ្ធភាពសម្រាប់បណ្ណាល័យឯកសារធំនៅពេលតែមួយ
- **ការអាប់ដេតបន្ថែមជាលំដាប់**: ការបន្ថែមឯកសារនិងការកែប្រែ​ក្នុងពេលពិត
- **ការគ្រប់គ្រងកំណែ**: ការកំណត់កំណែឯកសារ និងការតាមដានការផ្លាស់ប្ដូរ
- **ការគ្រប់គ្រងម៉េទាដេតា**: គុណលក្ខណៈឯកសារជាច្រើន និងតាក់សូណូមី

### លំនាំស្វែងរក និងយកទិន្នន័យ
- **ស្វ័យប្រវត្តិសមុទ្រមួយ (Hybrid Search)**: ភ្ជាប់ការស្វែងរកដោយអត្ថន័យ និងពាក្យសម្រង់សម្រាប់លទ្ធផលល្អបំផុត
- **ស្វែងរកតាមជាតិសម្បក (Faceted Search)**: ការត្រង និងចាត់តម្លៃក្នុងវិមាត្រច្រើន
- **កែម៉ោងសុពលភាព (Relevance Tuning)**: អាលហ្គរីធម์ពិន្ទុប្តូរតាមដើម្បីសមនឹងដែនវិស័យ
- **ចំណាត់ថ្នាក់លទ្ធផល**: ការកែចម្ការរើសដោយថ្នាក់ជាមួយតុល្យកម្មលក្ខណៈអាជីវកម្ម

### លំនាំសុវត្ថិភាព
- **សុវត្ថិភាពនៅលើកម្រិតឯកសារ**: ការគ្រប់គ្រងចូលប្រើលម្អិតសម្រាប់ឯកសារតែមួយៗ
- **ចាត់ថ្នាក់ទិន្នន័យ**: ការកំណត់សម្គាល់ផ្ទាល់ខ្លួននៃភាពងាយរងគ្រោះ និងការការពារ
- **ស្នាមតាមសិក្សា (Audit Trails)**: ការកត់ត្រាយ៉ាងទូលំទូលាយនៃប្រតិបត្តិការទាំងអស់ RAG
- **ការការពារឯកជនភាព**: ការរកឃើញ PII និងការកាត់ឯកសារចេញ
